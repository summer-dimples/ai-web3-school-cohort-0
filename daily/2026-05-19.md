### Key Learnings

**LLM**
1. LLM 是概率预测机器，不是"理解"语言，而是学习 token 之间的统计关系。自回归生成：每次预测一个 token，追加到上下文，继续预测。
2. 擅长：语言理解、代码生成、推理；不擅长：精确事实（幻觉）、跨会话记忆、确定性计算。

**Prompt**
3. System Prompt 设置身份和边界，User Message 传递任务意图。少样本（Few-shot）和思维链（CoT）是最常用的提升质量技巧。
4. Prompt ≠ 安全保障。关键操作必须在代码层面加验证，不能只靠 prompt 约束模型行为。

**Context**
5. 上下文窗口是模型的"工作内存"，超出窗口的内容模型完全不可见。"Lost in the Middle"现象：关键信息放中间容易被忽略，应放开头或结尾。

**RAG**
6. RAG 核心流程：Query → Embedding → 向量搜索 → 注入上下文 → 生成。解决知识截止日期问题和专有知识幻觉问题。
7. RAG vs 微调：知识动态变化用 RAG；改变模型行为/风格用微调。

**Agent**
8. Agent = 自主规划 + 动态工具调用 + 跨轮状态管理。基础模式是 ReAct Loop：观察 → 思考 → 行动 → 获取结果 → 循环。
9. **Web3 关键**：AI Agent 不应直接接触私钥/助记词；签名、转账、合约写入必须保留人工确认节点。

**Frameworks**
10. LangGraph 适合有状态有循环的复杂 agent；OpenAI Agents SDK 适合多 agent handoff；Hermes 适合长期运行和跨 session。本课程 Day 0 已配置 Hermes + GLM-5.1。

**Vibe Coding**
11. Vibe Coding 改变了工作重心：从写代码 → 描述意图 + 审查代码。架构决策、安全审查、业务逻辑验证仍必须由人主导。
12. AI 输出的 5 类风险：事实错误、引用错误、推理漂移、执行越权、工具误用。Web3 合约代码必须审查后才能部署。

**MCP**
13. MCP 是 Anthropic 的开放标准，统一了 AI 模型连接外部工具/数据的方式。三个原语：Resources（数据）/ Tools（动作）/ Prompts（模板）。
14. 安全原则：只读操作可让 AI 自动调用；写入操作（签名、转账、合约部署）必须人工确认。
