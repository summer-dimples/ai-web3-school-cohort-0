# LLM 大语言模型

## 核心机制：它在做什么？

LLM 本质上是一个**概率预测机器**：给定一段文本（上下文），预测下一个最合理的 token 序列。

```
输入：「今天天气很」
预测：「好」「热」「冷」「晴」...（各有概率权重）
输出：按概率采样选一个 → 追加到上下文 → 继续预测下一个
```

这个过程叫**自回归生成（autoregressive generation）**。模型不是"理解"文字，而是学习了海量文本中 token 之间的统计关系。

**Token 不是字符**：`"Hello"` 可能是 1 个 token，`"你好"` 可能是 2 个 token，`"ai-web3"` 可能被切成 `["ai", "-", "web", "3"]`。token 数决定了 API 费用和上下文长度消耗。

## 训练过程（简化版）

1. **预训练**：在互联网规模的文本上预测下一个 token，学到语言结构和世界知识
2. **指令微调（SFT）**：在人工标注的问答对上继续训练，让模型学会"有用的回答风格"
3. **RLHF**：人类对模型输出打分，强化学习让模型输出更受偏好的内容

## LLM 擅长什么 / 不擅长什么

| 擅长 | 不擅长 |
|------|--------|
| 语言理解与生成 | 精确事实记忆（会"幻觉") |
| 代码生成与解释 | 确定性计算（数学计算会出错）|
| 文本摘要与翻译 | 跨会话状态保持（无持久记忆）|
| 推理与规划（辅助） | 知识截止日期后的事件 |
| 格式转换与结构化输出 | 感知物理世界 |

## 关键参数

| 参数 | 含义 | 常用值 |
|------|------|--------|
| `temperature` | 输出随机性。0 = 确定，1 = 创意，>1 = 混乱 | 0（代码）/ 0.7（对话）|
| `max_tokens` | 最大输出 token 数 | 按需设置 |
| `top_p` | 核采样，只从累积概率≥p 的 token 中选 | 0.9 |
| `model` | 选哪个模型 | claude-sonnet-4-6 / gpt-4o |

## 幻觉（Hallucination）

模型"自信地说错话"。原因：模型优化的是"听起来合理"，不是"确保准确"。

**常见场景：**
- 编造不存在的论文、链接、API
- 混淆相近概念或时间
- 对训练数据不足的领域瞎猜

**应对方法：**
- 关键事实务必外部核实
- 不信任模型提供的 URL 和引用
- 提供上下文让模型"查"而非"猜"（→ RAG）

## API 最小调用示例

```python
from anthropic import Anthropic

client = Anthropic()
response = client.messages.create(
    model="claude-sonnet-4-6",
    max_tokens=1024,
    messages=[{"role": "user", "content": "解释什么是 token"}]
)
print(response.content[0].text)
```
