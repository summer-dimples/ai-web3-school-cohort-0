# Web3 密码学 + 钱包基础

## 用 AI 视角理解密码学

AI 系统依赖"身份验证 + 数据完整性"，Web3 也一样，只是用密码学原语实现：

| AI 类比 | Web3 对应 | 用途 |
|---------|----------|------|
| API Key | 私钥（Private Key）| 证明你是你 |
| 请求签名 | 数字签名 | 证明这个操作是你授权的 |
| 哈希 ID | 交易哈希（Tx Hash）| 唯一标识一笔操作 |
| 数据库唯一 ID | 区块哈希 | 唯一标识一个区块 |

---

## 哈希（Hash）

**核心特性：**
- **单向**：给定输入可算出输出，反向不可能
- **确定性**：同样输入永远输出同样结果
- **雪崩效应**：改一个字符，输出完全不同
- **固定长度**：不管输入多长，SHA-256 输出都是 256 位

```
"hello"  → SHA-256 → 2cf24dba5fb0a30e26e83b2ac5b9e29e...
"Hello"  → SHA-256 → 185f8db32921bd46d35cc4ac6... （完全不同）
```

**在 Web3 中的用途：**
- 交易哈希：每笔交易的唯一指纹
- 区块哈希：链式结构的核心（每个块包含上一个块的哈希 → 改了一块就要改所有后续块 → 不可篡改）
- 数据承诺：Merkle Tree 的基础

## 公私钥对

**生成流程：**
```
随机数（256位）→ 私钥（Private Key）
                    ↓ 椭圆曲线乘法（secp256k1）
                公钥（Public Key）
                    ↓ Keccak-256 哈希 + 取后20字节
                地址（Address，0x 开头，42位）
```

**关键性质：**
- 私钥 → 公钥 → 地址：**单向，不可逆**
- 知道地址，无法推出公钥；知道公钥，无法推出私钥
- 地址是公开的（类似银行卡号），私钥是秘密的（类似密码）

**安全直觉（对比 AI 场景）：**
- API Key 泄露 → 额度被盗用，可以换 Key
- 私钥泄露 → 资产永久丢失，无法撤销，**没有"重置密码"**

## 数字签名

**签名过程：**
```
交易内容 + 私钥 → 签名算法 → 数字签名
                                  ↓
                              任何人用公钥验证 → 确认这个签名确实来自私钥持有者
```

**关键理解：**
- 签名 ≠ "点一下确认"，签名 = **授权一个具体的链上动作**
- 每次签名都会暴露你同意了什么（金额、接收方、合约调用）
- 不同签名类型的风险不同：

| 类型 | 风险 | 说明 |
|------|------|------|
| 读取数据 | 无风险 | 不需要签名 |
| 消息签名 | 中风险 | 可能被钓鱼网站滥用 |
| ETH 转账 | 高风险 | 直接资产转移 |
| 合约授权（approve）| 极高风险 | 可能授权无限额度 |

**AI Agent 与签名的边界：**
- Agent 可以：解释交易内容、准备参数、检查风险
- Agent 不可以：代替你签名、保管私钥

## Merkle Tree

**用来干什么：** 高效验证大量数据中某条数据的存在性，不需要下载全部数据。

```
          Root Hash
         /         \
    Hash(AB)     Hash(CD)
    /    \        /    \
Hash(A) Hash(B) Hash(C) Hash(D)
  A       B       C       D
```

**验证 A 存在**：只需要 Hash(B) + Hash(CD) + Root Hash，不需要 C、D 本身。

**Web3 应用：**
- 区块验证（轻客户端）
- NFT/Token 空投白名单验证（Merkle Proof）
- 状态证明

## 钱包基础

### 钱包不是账号，是**私钥管理器 + 签名工具**

```
钱包的核心功能：
1. 安全存储私钥（不让私钥离开设备）
2. 构造并签名交易
3. 与 DApp 交互（WalletConnect / MetaMask 注入）
```

### 助记词（Mnemonic）

- 12 或 24 个英文单词，按 BIP-39 标准
- 从助记词可以推导出**所有账户**的所有私钥
- **助记词 = 所有资产的主控权**

```
安全规则（不可违反）：
✓ 手写纸质备份，存多个安全位置
✗ 不截图
✗ 不存在手机/电脑
✗ 不上传云盘
✗ 不发给任何人（包括"客服"）
✗ 不输入到任何网页（包括"官方"网站）
```

### EOA vs Smart Account

| 特征 | EOA（外部账户）| Smart Account（智能账户）|
|------|--------------|------------------------|
| 创建方式 | 直接生成私钥 | 部署合约 |
| 控制方式 | 单一私钥 | 代码逻辑（多签、社交恢复）|
| 私钥丢失 | 资产永久丢失 | 可社交恢复 |
| Gas 支付 | 只能用 ETH | 可用其他 Token（Paymaster）|
| 批量操作 | 每步单独签名 | 一次签名完成多步 |
| 复杂度 | 简单，兼容性最好 | 复杂，功能强大 |

**Week 1 实践用 EOA 即可（MetaMask 默认类型）。**
