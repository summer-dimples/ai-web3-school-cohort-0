## 什么是 Agent

Agent 是一个能**自主规划、动态调用工具、跨轮管理状态**的 AI 系统。区别于单次 Prompt，Agent 的核心是"模型在决策环里"。

```
Prompt   → 人提问 → 模型回答 → 人决定下一步
Workflow → 预定义流程 → 模型是其中一个步骤 → 路径固定
Agent    → 模型自主规划 → 调用工具 → 根据结果决定下一步 → 循环
```

## Agent 的基本运行循环

```
┌─────────────────────────────────────────┐
│              Agent Loop                  │
│                                          │
│  观察（Observe）                          │
│      ↓                                  │
│  思考/规划（Think/Plan）                  │
│      ↓                                  │
│  行动（Act）→ 调用工具                    │
│      ↓                                  │
│  获取结果 → 回到观察                      │
│      ↓                                  │
│  目标达成 → 停止                          │
└─────────────────────────────────────────┘
```

这个模式叫 **ReAct（Reasoning + Acting）**，是当前主流 agent 框架的基础。

## Agent 核心技术组件

### 状态管理（State）
多个节点共享读写同一个 State 对象，跨步骤传递信息：

```python
state = {
    "task": "部署合约到 Sepolia",
    "wallet": "0x1234...",
    "step": 1,
    "artifacts": []
}
```

### 工具调用（Tool Calling）
模型输出结构化的工具调用请求，框架执行后把结果回传：

```json
{
  "tool": "deploy_contract",
  "params": {
    "bytecode": "0x608060...",
    "network": "sepolia"
  }
}
```

**关键**：模型不直接执行，框架执行并把结果返回给模型，模型再决定下一步。

### 长期记忆（Memory）
- **短期记忆**：当前上下文窗口（会话内）
- **长期记忆**：向量数据库 / 结构化存储（跨会话）

### MCP（Model Context Protocol）
LLM 与外部工具的统一连接协议。见 `week1-mcp.md`。

### Guardrails（护栏）
输入/输出验证规则，不符合则中止或转人工：

```python
# 示例：检测到涉及私钥操作，强制暂停
if "private_key" in tool_params:
    raise GuardrailViolation("涉及私钥操作，需要人工确认")
```

### Tracing（可观测性）
记录 agent 每一步的输入、输出、工具调用、耗时，用于调试和审计。

### Handoff（移交）
子任务完成后把控制权移交给另一个 agent 或人工。

### 错误恢复
执行失败时的策略：重试 → 降级 → 人工介入。

## 什么时候需要 Agent？

**适合用 Agent 的信号：**
- 目标开放式，无法提前写死步骤
- 需要多工具协作（搜索 + 代码生成 + 部署）
- 中间结果决定下一步策略
- 需要跨会话记忆状态

**不适合，用简单方案：**
- 一次性问答 → Prompt
- 流程固定 → 脚本/Workflow
- 高合规要求 → 人工审核节点
- 数据确定性要求高 → 直接数据库查询

**复杂度和风险越高，越要警惕过度 agent 化。**

## Agent 的风险

| 风险 | 描述 | 应对 |
|------|------|------|
| 执行越权 | 做了超出授权的操作 | Guardrails + 最小权限 |
| 工具误用 | 调用错误工具或参数 | Tracing + 人工审查 |
| 推理漂移 | 长上下文中逻辑断裂 | 分段验证 + 状态复位 |
| Prompt 注入 | 工具返回结果中含恶意指令 | 对工具结果做清洗 |
| 无限循环 | Agent 无法停止 | 最大步数限制 |

## AI × Web3 中的 Agent 特殊风险

**AI Agent 不应直接接触私钥 / 助记词。**

涉及以下操作时，**必须有人工确认节点**：
- 签名（授权具体动作）
- 转账（资产移动）
- 合约写入（状态变更，不可逆）
- 权限变更（授权范围扩大）

```
Agent 规划 → [人工确认] → 钱包签名 → 链上执行
```
